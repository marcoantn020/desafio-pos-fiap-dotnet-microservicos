Escopo das Promoções — Qual modelo?

O PDF exige que Admin possa "criar promoções". Como você quer modelar isso?

A

Desconto Percentual por Jogo

Admin cria uma promoção com título, percentual de desconto (ex: 20%) e validade. A promoção é associada a um ou mais jogos. Na listagem de jogos, o preço promocional aparece ao lado do original.

Pontos fortes

  • Mais rico e realista
  • Demonstra relação entre entidades
  • DDD claro (Promotion é Aggregate)

Pontos fracos

  • Mais código para implementar
B

Promoção Global Simples

Admin cria uma promoção com título, descrição, percentual de desconto e validade. Não vincula a jogos específicos — é um "aviso de promoção" que o sistema exibe.

Pontos fortes

  • Simples e rápido de implementar
  • Atende o requisito com menos código

Pontos fracos

  • Menos rico (sem vínculo com jogos)
C

Cupom de Desconto

Admin cria códigos de cupom (ex: FIAP20) com percentual de desconto e data de expiração. Usuário informa o código ao adicionar um jogo à biblioteca.

Pontos fortes

  • Funcionalidade diferenciada
  • Demonstra lógica de negócio rica

Pontos fracos

  • Maior complexidade
  • Requer mudança no fluxo da biblioteca
