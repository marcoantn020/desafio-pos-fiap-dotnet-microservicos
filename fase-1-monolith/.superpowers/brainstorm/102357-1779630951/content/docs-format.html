Formato da Documentação DDD

O PDF exige: Event Storming dos fluxos + Diagramas DDD conforme disciplina. Qual formato?

A

DrawIO (.drawio)

Arquivos XML abríveis no draw.io / diagrams.net (gratuito, online ou desktop). Gero os diagramas prontos como arquivos .drawio em /docs. Você abre, ajusta se quiser e exporta como PNG/PDF.

Pontos fortes

  • Editável visualmente
  • Amplamente reconhecido
  • Export para PNG/PDF fácil
  • Pode usar no vídeo de entrega

Pontos fracos

  • Requer draw.io para editar
B

Mermaid em Markdown

Diagramas como código Mermaid dentro de arquivos .md em /docs. Renderizam automaticamente no GitHub e no VS Code (com extensão).

Pontos fortes

  • Versionado junto com o código
  • Renderiza no GitHub automaticamente
  • Sem ferramenta externa

Pontos fracos

  • Menos visual para apresentação
  • Mermaid tem limitações para Event Storming
C

Ambos: DrawIO + Markdown

Arquivos .drawio para os diagramas visuais (Event Storming, Context Map) e um README em Markdown explicando cada diagrama com o modelo conceitual em texto.

Pontos fortes

  • Mais completo e profissional
  • DrawIO para apresentar, Markdown para referência
  • Melhor nota na avaliação DDD

Pontos fracos

  • Mais trabalho de geração
